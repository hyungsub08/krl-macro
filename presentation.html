02. 역공학 과정

1단계: 트래픽 캡처

mitmproxy로 코레일 앱의 HTTP 트래픽 가로채기. smart.letskorail.com에 TLS 피닝 미적용되어 캡처 가능. x-dynapath-m-token 헤더와 Sid 파라미터 확인.

2단계: APK 디컴파일

jadx/apktool로 코레일 앱 디컴파일. DynaPath SDK 클래스 식별. 인코딩 테이블, AES 키, 알고리즘 추출.

3단계: Python 재구현

DynaPathMasterEngine 클래스로 토큰 생성 로직 재구현. 매 요청마다 유효한 토큰 자동 생성.

4단계: 검증

로그인, 조회, 예매, 취소 전 과정에서 MACRO ERROR 없이 정상 동작 확인.

02. DynaPath 토큰 구조

매직 프리픽스bEeEP
길이 표시X
키 인코딩 (기본 테이블)v1+AB3X+1711234568123
본문 인코딩 (커스텀 테이블)ai=com.korail.talk&di=...&ts=...&dm=SM-S928N

토큰 생성 과정

  • 랜덤 4자리 + 타임스탬프로 동적 키 생성
  • 키를 62문자 테이블로 base-30 인코딩
  • 키에서 정수값을 파생하여 커스텀 테이블 생성
  • 디바이스 메타데이터를 커스텀 테이블로 인코딩
  • 매 요청마다 다른 토큰 생성

Sid 생성

plaintext = f"AD{timestamp_ms}" key = iv = b"2485dd54d9deaa36" Sid = AES_CBC(plaintext, key) Sid = base64(Sid) + "\n"

AES 키 16바이트가 앱 내에 하드코딩되어 있으며, 키와 IV가 동일합니다.

02. 우회가 가능했던 근본 원인

1

클라이언트 측 토큰 생성

서버 참여 없이 클라이언트만으로 토큰 생성 가능. 로직을 알면 누구나 유효한 토큰 생성.

2

하드코딩된 비밀값

인코딩 테이블, AES 키, 앱 서명 등이 APK에 하드코딩. 디컴파일로 즉시 추출.

3

앱 무결성 미검증

Play Integrity API 등 외부 검증 없음. su=false, emu=false 등 자체 보고값은 위조 가능.

4

TLS 피닝 미적용

메인 API 서버에 인증서 피닝 미적용. mitmproxy로 트래픽 캡처 가능.

5

고정 알고리즘

앱 업데이트 없이 알고리즘 변경 불가. 한 번 역공학하면 장기간 유효.

6

서버 측 심층 검증 부재

토큰 형식만 검증. 타임스탬프 유효 범위, device_id 실존 여부 미검증.

05. 핵심 방어 대책 상세

TLS 핑거프린팅

Python requests의 TLS 핸드셰이크 패턴은 실제 Android 기기와 다릅니다. User-Agent는 Android 13이지만 JA3 핑거프린트는 Python/urllib3의 것.

서버 측만으로 즉시 탐지 가능

App Attestation

Play Integrity API로 앱 바이너리 무결성과 실행 환경을 Google 서버에서 직접 검증. 역공학된 Python 클라이언트는 유효한 attestation을 생성할 수 없음.

가장 근본적인 해결책

Challenge-Response

서버가 매 요청마다 고유 nonce를 발급. 클라이언트가 nonce를 포함하여 토큰 생성. 클라이언트 독립 토큰 생성 불가.

DynaPath 근본 취약점 해결

대기열 시스템

취소표 발생 시 즉시 공개하지 않고 30초간 모인 요청 중 랜덤 배정. 속도 우위를 원천적으로 무력화.

매크로 존재 의미 자체를 제거

08. 차단 시 진단 가이드

증상별 원인 분석 및 즉시 대응 방법

증상원인즉시 대응성공률

08. 대체 자동예매 방법

현재 방식 차단 시 대안 — 난이도/성공률 순위

1. 간격/패턴 조정

난이도 하 성공률 높음

랜덤 지터(±1초) 추가, 가우시안 분포 간격. 코드 수정만으로 즉시 적용.

2. TLS 에뮬레이션

난이도 중 성공률 높음

curl_cffi로 실제 Chrome TLS 핑거프린트 모방. requests 교체로 적용.

3. 브라우저 자동화

난이도 중 성공률 중~높

Playwright로 코레일 웹 직접 제어. DynaPath 무관. 속도 느림.

4. IP 로테이션

난이도 중 성공률 중

주거용 프록시 또는 모바일 CGNAT로 IP 분산. IP 차단에 효과적.

5. 실제 기기 자동화

난이도 상 성공률 높음

ADB + UIAutomator2로 공식 앱 조작. 모든 차단 우회. 기기 필요.

6. API 역공학 재수행

난이도 상 성공률 변동

최신 APK 디컴파일 → DynaPath 변경 분석. Challenge-Response 시 무력.

권장 대응 순서: 간격 조정 → TLS 에뮬레이션 → 브라우저 자동화 → 실제 기기
